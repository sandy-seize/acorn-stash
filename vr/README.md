# VR법 운용 비서 🧮

밸류 리밸런싱(VR법)을 **매주 "오늘 얼마 사고/팔아라"** 지시로 바꿔주는 실전 운용 보조 도구.
백테스터가 아니라 **실전 비서**가 목적이다.

라이브: https://acorn-stash.vercel.app/vr

---

## 현재 구현 (v1)

### 기능
- **설정** — 종목 코드/티커, 총 원금(풀), 분할 수 T(기본 40), V값 증가 방식(정액/정률), 리밸런싱 밴드
- **자동 시세 조회** — 국내(Toss) / 미국(Yahoo)
- **주간 리밸런싱 지시** — 목표 V값 대비 평가액을 비교해 **매수 / 매도 / 관망**과 정확한 금액·수량 제시
- **체결 기록** — 실제 체결가·수량을 입력하면 보유수량·평단·현금·손익·거래로그 자동 갱신, 주차 +1
- **포지션 저장** — 브라우저 `localStorage`(`vr_state_v1`), 개인·단일 기기

### VR 엔진 수식
```
unit          = seed / T                      # 1회(=매주) 목표 증가 단위
targetV(week) = min(unit * week, seed)         # 정액(기본)
              = min(unit * (1+g)^(week-1), seed)# 정률
평가액 < V*(1-band)  → 매수 (V까지 채움, 현금 한도)
평가액 > V*(1+band)  → 매도 (초과분만큼)
그 외                → 관망
```
> 이 수식은 SOXL 실데이터 백테스트(강세장 +249.6%)와 **정확히 일치**함을 Node로 교차검증함.

### 파일 구조
```
vr/index.html   — 앱 (정적 HTML 한 장, 핑크 도토리 디자인)
api/price.js    — 시세 프록시 (Vercel 서버리스, acorn-stash 최초의 함수)
```

### 데이터 소스 — 왜 프록시가 필요한가
- Toss 비공식 가격 API(`wts-info-api.tossinvest.com/api/v2/stock-prices`)는 **브라우저 직접 호출 시 CORS 403**으로 차단됨.
- 따라서 서버사이드 프록시 `api/price.js`를 경유한다.
  - `?code=&market=`
  - 국내(A코드/6자리 숫자) → **Toss** (가격 + 종목명)
  - 미국(티커) → **Yahoo** chart meta
- 미국 종목은 Toss 검색 엔드포인트를 확보하지 못해 Yahoo로 우회 중.

### 검증 내역
- 프록시: 삼성전자·카카오(Toss), SOXL·AAPL(Yahoo), 빈코드/없는종목 에러까지 정상
- VR 엔진: 백테스트 결과 재현(34,960,121원 / +249.6%)
- 로컬 미리보기(미니 Node 서버)로 페이지 + 프록시 end-to-end 동작 확인

## 로드맵 — 저관여자도 쓸 수 있게 (다음 단계)

> **문제의식:** 지금은 사용자가 종목 코드/티커를 직접 입력해야 한다.
> 주식을 모르는 **저관여 사용자는 "뭘 사야 하는지"부터 막힌다.**
> → **Toss가 제공하는 데이터를 활용해 "종목을 모르는 사람도 시작할 수 있게" 기능을 고도화한다.**

### 핵심 방향
1. **종목 검색/자동완성** — 이름·초성으로 검색 (예: "삼성" → 삼성전자/삼성SDI…), Toss 검색 데이터 연동
2. **큐레이션 진입점** — 코드 입력 대신 **고르기**로 시작
   - 인기/거래대금 상위 종목
   - 테마·섹터(반도체, 2차전지, 빅테크 ETF…)
   - VR법에 어울리는 **변동성 큰 ETF** 추천 묶음(저관여자가 개별주 대신 분산 ETF로 시작)
3. **온보딩 가이드** — VR법이 뭔지, 왜 분할/리밸런싱하는지 1~2스텝 설명 후 바로 설정
4. **추천 기본값** — 원금만 넣으면 T·밴드·증가방식을 안전한 기본값으로 자동 세팅(고급설정은 접어두기)
5. **리스크 고지 강화** — 레버리지 ETF 위험, "보험이지 수익극대화 아님" 톤 유지

### 선행 과제 (기술)
- Toss **검색/자동완성 엔드포인트** 발굴(현재 미확보) 및 프록시에 라우트 추가
- Toss **미국주식 코드 체계** 확보 시 미국도 Toss로 통일(현재 Yahoo 우회)
- 큐레이션 목록의 데이터 출처/갱신 방식 결정(정적 시드 vs Toss 랭킹 API)

## 한계 / 주의
- v1은 **단일 종목·단일 기기**(멀티 종목·기기 동기화 미지원)
- 시세는 **참고용**, 실제 체결가와 다를 수 있음
- Toss는 **비공식 엔드포인트**라 변경 시 깨질 수 있음 → 프록시 계층에서 흡수/대체 가능하도록 격리
- 투자 판단과 책임은 전적으로 사용자 본인에게 있음
